# PAPER 1: SECTION A (COMPULSORY)

## QUESTION ONE (30 MARKS)

### a) Explain the differences between:

#### i. Concurrency and Persistence (4 Marks)

* **Concurrency:** The ability of a system to execute multiple operations or tasks at the same time without them interfering with each other.
* **Persistence:** The ability of an object to survive a system shutdown or process termination, ensuring its state is saved permanently (e.g., in a database).

#### ii. Encapsulation and Abstraction (4 Marks)

* **Abstraction:** The practice of hiding complex implementation details and showing only the essential features of an object (the *"what it does"*).
* **Encapsulation:** The practice of wrapping data (variables) and code (methods) together into a single unit (a class) and restricting direct access to the data (the *"how it hides"*).

---

### b) Discuss any four primary tasks in Object-Oriented Analysis (OOA) (4 Marks)

1. **Identify the Objects/Classes:** Find the key real-world entities within the system domain.
2. **Define Attributes:** Identify the data properties associated with each class.
3. **Define Operations/Methods:** Determine what behaviors or actions each class can perform.
4. **Establish Relationships:** Define how the classes interact and connect with each other (e.g., inheritance, association).

---

### c) Describe the purpose of the following as used in OOAD: (6 Marks)

#### i. UML Sequence Diagram

* **Purpose:** A dynamic diagram that shows how objects interact with one another over time, emphasizing the chronological order in which messages are exchanged during a specific scenario.

#### ii. UML Class Diagram

* **Purpose:** A static structural diagram that describes the system's structure by showing its classes, their attributes, methods, and the relationships among them.

---

### d) Outline any three major elements in an object-oriented system (6 Marks)

1. **Classes:** The blueprints or templates used to create individual objects.
2. **Objects:** The concrete, runtime instances of a class that hold actual data.
3. **Methods:** The functions or behaviors defined inside a class that operate on its data.

---

### e) Discuss any three benefits of using the object model (6 Marks)

1. **Reusability:** Code written for one project can easily be reused in other systems through inheritance and modular classes.
2. **Maintainability:** Because code is broken down into isolated objects, fixing bugs or updating features is easier without breaking the whole system.
3. **Scalability:** It makes designing complex systems easier to manage by breaking large problems into smaller, independent chunks.

# PAPER 1: SECTION B (40 MARKS)

## QUESTION TWO (20 MARKS)

### a) Discuss the three levels of testing used in object-oriented systems (3 Marks)

1. **Unit Testing:** Testing individual methods or single classes in isolation to ensure they work correctly.
2. **Integration Testing:** Testing how a group of interacting classes cooperate and communicate with each other.
3. **System Testing:** Testing the complete, fully integrated software system as a whole to verify it meets all requirements.

---

### b) List the steps in visualization of object model (7 Marks)

1. Identify the core user requirements or use cases.
2. Extract the candidate conceptual classes from the problem statement.
3. Define the attributes and key data for each class.
4. Determine the behaviors and responsibilities (methods) of the classes.
5. Map the structural connections and relationships between classes.
6. Draw the initial UML structural diagrams (like Class Diagrams).
7. Refine and validate the model through dynamic behavior diagrams (like Sequence or Activity diagrams).

---

### c) Using a Car as an example, discuss the three main characteristics of an Object. (10 Marks)

1. **State (Attributes):** The data or characteristics that an object holds at a given moment.
* *Car Example:* A specific car's state includes its color (Blue), current speed (60 km/h), fuel level (80%), and gear (4th).


2. **Behavior (Methods):** The actions or operations that an object can perform or undergo.
* *Car Example:* A car can execute actions like `accelerate()`, `applyBrakes()`, `turnLeft()`, and `switchEngineOff()`.


3. **Identity (Unique Address):** The property that distinguishes an object from all other objects in the system, even if their states are identical.
* *Car Example:* Two identical blue Toyota cars made in the same year have unique identities, distinguished by their unique engine numbers or memory addresses in a program.

## QUESTION THREE (20 MARKS)

### a) Explain the main difference between object-oriented analysis and other forms of analysis (2 Marks)

* **Object-Oriented Analysis (OOA)** organizes a system around **objects** that combine both data and behavior.
* **Traditional Analysis (like Structured Analysis)** separates the system into distinct **data** (data flows) and **processes** (functions that manipulate the data).

---

### b) Discuss any four parts of a state in a state transition diagram (8 Marks)

1. **State Name:** A clear label distinguishing the state (e.g., `Idle`, `Processing`).
2. **Entry Actions:** Operations executed automatically immediately upon entering the state.
3. **Exit Actions:** Operations executed automatically right before leaving the state.
4. **Internal Transitions:** Actions performed inside the state that do not cause a change to a different state.

---

### c) Development process of an object-oriented system involves four major phases.

#### i. List and discuss the four phases. (8 Marks)

1. **Analysis (OOA):** Understanding the domain and building a model of *what* the system needs to do using real-world objects.
2. **Design (OOD):** Taking the analysis model and defining *how* it will be built, specifying software classes, interfaces, and architecture.
3. **Programming/Implementation:** Writing the actual code in an object-oriented language (e.g., Java, C++) based on the design models.
4. **Testing:** Verifying that the objects interact correctly and the final application meets customer expectations.

#### ii. Explain why its advisable for development process to go through the four phases. (2 Marks)

* Going through these phases systematically reduces project failure risk. It ensures developers fully understand user needs before coding, resulting in a cleaner, well-structured, and bug-free system that is easy to maintain later.

### a) Discuss any two advantages and two disadvantages of object oriented programming (4 Marks)

* **Advantages:**
1. *Code Reusability:* Through inheritance, you don't have to rewrite code.
2. *Easy Maintenance:* Modular code makes finding and fixing bugs highly isolated.


* **Disadvantages:**
1. *Steep Learning Curve:* Thinking in terms of strict object relationships can be difficult for beginners compared to procedural programming.
2. *Larger Program Size:* OOP applications often require more lines of initial boilerplate code and memory overhead.

4. **Coding/Implementation:** Translating the blueprints into functional code.

### f) & g) Implement the provided Circle class design into code (3 marks)

Here is the clean implementation using **Java**:

```java
public class Circle {
    // Private attributes (- means private, # means protected)
    private double xCoord;
    private double yCoord;
    protected double radius; // '#' indicates protected

    // Constructor
    public Circle(double xCoord, double yCoord, double radius) {
        this.xCoord = xCoord;
        this.yCoord = yCoord;
        this.radius = radius;
    }

    // Public methods (+ means public)
    public double findArea() {
        return Math.PI * radius * radius;
    }

    public double findCircumference() {
        return 2 * Math.PI * radius;
    }

    public void scale(double factor) {
        this.radius = this.radius * factor;
    }
}

```

### c) Discuss Robustness Analysis. Identify the roles of entity objects, control objects, and interface objects. How do these objects contribute to a robust object-oriented system? (10 Marks)

#### What is Robustness Analysis?

It is an analytical approach used to validate use-case descriptions against structural possibilities, sorting software components into three distinct conceptual blocks: **Boundary (Interface)**, **Entity**, and **Control** objects.

#### Roles of the Objects:

1. **Interface/Boundary Objects:** Elements that interact directly with actors outside the system boundaries (e.g., user screens, login UI windows, API endpoints).
2. **Entity Objects:** Objects representing core, long-lived system data, usually mapped straight from database tables (e.g., `UserAccount`, `Invoice`).
3. **Control Objects:** The "brains" or business logic handlers that connect the boundary interfaces to the data entities. They coordinate workflows and manage data manipulation steps (e.g., `LoginController`).

#### Contribution to System Robustness:

* By separating concerns completely, your system becomes highly modular. If the user interface changes (Boundary), your business logic rules (Control) and database structure (Entity) stay safe and completely unaffected. This keeps the application incredibly stable and adaptable to alterations.
