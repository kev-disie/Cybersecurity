# PAPER 1: MAIN EXAMINATION (Dis1.jpeg & Dis2.jpeg)

## SECTION A

### QUESTION ONE

**a) Define distributed systems. (2 Marks)**
A distributed system is a collection of independent computers that appear to its users as a single, unified coherent system.

**b) Discuss five characteristics of a distributed system. (5 Marks)**

1. **Resource Sharing:** Users can easily share hardware, software, or data across the network.
2. **Concurrency:** Multiple tasks run at the same time on different machines independently.
3. **No Global Clock:** Nodes do not share a single, absolute central timer; coordination relies on message exchanges.
4. **Independent Failures:** If one computer crashes, the rest of the system can keep running.
5. **Heterogeneity:** The system works across different operating systems, hardware platforms, and networks.

**c) Explain any four dimensions of transparency in distributed system. (4 Marks)**

1. **Access Transparency:** Hides differences in data representation and how a resource is accessed (e.g., using local vs. remote files identically).
2. **Location Transparency:** Hides where a resource is physically located in the network.
3. **Migration (Mobility) Transparency:** Hides that a resource can move to another physical location while in use.
4. **Replication Transparency:** Hides the fact that multiple copies of a resource exist for backup or speed.

**d) State what you understand by transparency as far as distributed systems are concerned. (2 Marks)**
Transparency is the ability of a distributed system to hide its distribution complexities, making the entire network look and feel like a single local system to the end-user.

**e) Discuss two advantages and two disadvantages of sharing resources in a distributed system. (4 Marks)**

* **Advantages:**
1. *Cost Efficiency:* Expensive hardware (like printers or high-end storage) can be shared by many users.
2. *Collaboration:* Teams can easily work together on shared databases and files from different places.


* **Disadvantages:**
1. *Security Vulnerabilities:* Shared access points increase the risk of unauthorized data leaks or hacks.
2. *Performance Bottlenecks:* If too many nodes try to access the same shared resource simultaneously, system performance slows down.



**f) Discuss two techniques employed to implement a service directory. (4 Marks)**

1. **Iterative Resolution:** A client asks a directory server for a location. If that server doesn't know, it sends back the address of another server. The client then manually contacts the next server itself.
2. **Recursive Resolution:** A client asks a directory server. If that server doesn't know, it contacts the next server *on behalf of* the client, gathers the answer, and passes it back down to the client.

**g) List and discuss any two mechanisms that can be used to ensure performance in distributed systems. (4 Marks)**

1. **Caching:** Storing copies of frequently used data closer to the client (in local memory) to avoid slow network fetches.
2. **Replication:** Placing copies of data on multiple servers across the network to distribute the user load and reduce travel distance for requests.

**h) Explain the meaning of "Remote IPC" and describe the four functions of Remote IPC. (5 Marks)**

* **Meaning:** Remote Inter-Process Communication (IPC) refers to the mechanisms that allow processes running on different, physically separated machines to exchange data and messages over a network.
* **Four Functions:**
1. *Message Passing (Send/Receive):* Directly moving raw data blocks from one node's memory space to another.
2. *Marshalling/Serialization:* Packing complex data structures into a flat, byte-stream format suitable for network transmission.
3. *Unmarshalling/Deserialization:* Unpacking the byte-stream back into its original data structures on the receiving end.
4. *Synchronization:* Coordinating the timing between the sender and receiver so messages aren't lost or read prematurely.

## SECTION B

### QUESTION TWO

**a) (i) Explain two features of a good distributed file system. (2 Marks)**

1. **High Availability:** The files should remain accessible even if a storage server crashes or a network link fails.
2. **Location Independence:** File names should not change or reveal their actual physical storage path when moved across servers.

**(ii) State and explain two concurrency problems in distributed systems. (4 Marks)**

1. **Lost Update Problem:** Two transactions read the same data simultaneously, alter it, and write it back. The second write accidentally overwrites (erases) the updates made by the first.
2. **Inconsistent Read Problem:** A transaction reads data while another transaction is mid-way through updating it, causing the reading transaction to see an incomplete, inaccurate state of the system.

**b) (i) Distributed systems use distributed algorithms to ensure synchronization. Describe five properties of these algorithms. (5 Marks)**

1. Relevant information is distributed across multiple machines.
2. Machines make decisions based solely on locally available information.
3. A single point of failure must be avoided (no central coordinator machine).
4. There is no absolute, common global clock.
5. Outright failure of one node does not break the entire synchronization algorithm.

**(ii) Explain the role of SSL/TLS in securing communication over the internet. How does the handshake process work, and what are the key components involved? (9 Marks)**

* **Role:** SSL/TLS provides **Encryption** (prevents eavesdropping), **Authentication** (proves identity), and **Data Integrity** (prevents tampering) over untrusted networks.
* **Key Components:** Client, Server, Certificate Authority (CA) digital certificates, Public/Private keys, and Symmetric session keys.
* **Handshake Process:**
1. *Client Hello:* Client sends supported cryptographic settings and a random number to the server.
2. *Server Hello & Certificate:* Server responds with its chosen settings, its own random number, and its CA-signed Digital Certificate containing its Public Key.
3. *Authentication:* The client verifies the server’s certificate against trusted root authorities.
4. *Key Exchange:* The client generates a "Pre-Master Secret," encrypts it with the server's Public Key, and sends it to the server.
5. *Session Key Generation:* Both sides use the secret and random numbers to generate matching **Symmetric Session Keys**.
6. *Finished:* Both exchange encrypted messages confirming that future communication will be encrypted using these fast symmetric keys.

### QUESTION THREE

**a) Using example of a BANK, Identify and explain the ACID properties in line with bank transactions. (12 Marks)**

* **Atomicity ("All or Nothing"):** If you transfer $100 from Account A to Account B, money must be debited from A *and* credited to B. If the system crashes mid-way, the whole transaction rolls back so no money vanishes into thin air.
* **Consistency:** The total pool of money must remain valid. If Account A has $500 and B has $200 (Total = $700), after transferring $100, the sum of both accounts must still equal exactly $700.
* **Isolation:** If two separate people are depositing money into Account A at the exact same moment, the system processes them sequentially in isolation so that one deposit doesn't overwrite or skip the other.
* **Durability:** Once the bank system sends a "Transaction Successful" receipt, the update is permanently written to hard storage. Even if the power cuts out immediately after, the balance update is not lost.

**b) Failure masking in a Distributed System requires proper detection mechanisms. Discuss the importance of failure detection in this context. (6 Marks)**

1. **Prevents Resource Waste:** Stops healthy nodes from trying to communicate or route traffic through dead nodes.
2. **Triggers Replicas:** Allows the system to immediately point traffic to backup/cloned nodes before the user notices an error.
3. **Maintains Consistency:** Ensures dead nodes are removed from vote majorities in consensus protocols, preventing split-brain errors.

**c) Explain dependability and availability as used in distributed systems. (2 Marks)**

* **Dependability:** The overall measure of a system's reliability, safety, and ability to deliver service without breaking down over a given period.
* **Availability:** The specific percentage of time that the system is fully up, running, and accessible to perform tasks when needed (e.g., "99.9% uptime").

### QUESTION FIVE (From Dis2.jpeg)

**a) Differentiate between Class Diagram and Activity Diagram. (2 Marks)**

* **Class Diagram:** A **static structural** diagram that shows the system's blueprints (classes, attributes, and how they connect to each other).
* **Activity Diagram:** A **dynamic behavioral** diagram that shows the step-by-step workflow or procedural flow of activities in a system.

**b) Identify and explain the structural and behavioral aspects of objects in object-oriented design. Illustrate with an example of a Class Diagram and an Activity Diagram. (8 Marks)**

* **Structural Aspect:** Defines *what* an object is made of. It captures static characteristics via attributes (data fields) and relationships to other objects.
* **Behavioral Aspect:** Defines *what* an object does. It captures operations, methods, changes of state, and how objects interact over time.
* *Illustration concept:* A **Class Diagram** would draw a block named `BankAccount` with structural fields like `balance` and methods like `deposit()`. An **Activity Diagram** would draw a flowchart showing the behavior of withdrawing money: `[Enter Pin] -> [Check Balance] -> [Dispense Cash]`.

**c) Discuss Robustness Analysis. Identify the roles of entity objects, control objects, and interface objects. How do these objects contribute to a robust object-oriented system? (10 Marks)**

* **Robustness Analysis:** A bridge methodology that sits between *Use Case analysis* and *Detailed Design*. It acts as a sanity check to ensure the system design handles all real-world logical pathways cleanly.
* **Roles of the Objects:**
1. *Interface (Boundary) Objects:* Handle interaction between the system and external actors (e.g., web screens, API endpoints).
2. *Entity Objects:* Hold the core, long-lived data structures (e.g., Database tables, `User` profiles).
3. *Control Objects:* Act as the business logic engines, connecting Boundary actions to Entity modifications.


* **Contribution to Robustness:** By explicitly separating presentation (Interface), logic (Control), and data storage (Entity), developers can change one piece without breaking the others, vastly increasing modularity and system resilience.

### QUESTION 1

**a) Briefly explain three forms of redundancy as used in distributed systems. [6 Marks]**

1. **Hardware Redundancy:** Having extra physical components (like dual power supplies or backup servers) ready to take over if the main unit breaks.
2. **Software Redundancy:** Running identical processes on different machines simultaneously so if one software instance bugs out, the others provide the valid answer.
3. **Data Redundancy:** Keeping identical copies of databases/files across multiple disks or servers (e.g., RAID arrays or replicated databases).

**b) Describe three (3) strategies that are used to implement Scalability in Distributed Systems. [6 Marks]**

1. **Hiding Communication Delays:** Using asynchronous processing so the system continues working on other jobs while waiting for slow network replies.
2. **Distribution:** Splitting a massive dataset or component down into small, manageable fragments and spreading them across different nodes (e.g., partitioning a database).
3. **Replication/Caching:** Making multiple copies of data closer to users to handle massive traffic volume simultaneously without choking a single core server.

**c) Explain the meaning of the following terminologies and state their roles in Distributed Systems: [4 Marks]**

* **i. Mutual Exclusion:** A rule ensuring that only one process can access a shared resource or critical section of code at any single moment. *Role:* Prevents data corruption caused by simultaneous writes.
* **ii. Deadlocks:** A circular state where a set of processes are blocked because each process holds a resource the other needs, and no one can proceed. *Role:* Systems must actively track this to resolve freezes and keep tasks moving.

**d) Explain two (2) system administration tasks in distributed systems. [4 Marks]**

1. **Resource Monitoring:** Tracking CPU usage, network bandwidth, and disk health across all scattered nodes to catch failures early.
2. **Configuration/Patch Management:** Deploying identical software updates, security fixes, and configurations uniformly across all system machines.

### QUESTION 2

**a) Distributed systems security is fundamentally more complex to implement than in standalone systems. Identify and explain key challenges in Distributed System's security. [6 Marks]**

1. **Expanded Attack Surface:** With data traveling across open, public networks, there are far more points where hackers can intercept data.
2. **Perimeter Breakdown:** There is no single "wall" or perimeter to protect; security must be enforced on every single independent node.
3. **Trust Management:** Harder to securely verify the true identity of remote users or remote hardware nodes across varying external networks.

**b) Describe any three (3) types of deadlocks that can occur in a Distributed System. [6 Marks]**

1. **Resource Deadlock:** Nodes are stuck waiting for physical or logical assets (like a specific database row or memory block) held by one another.
2. **Communication Deadlock:** A circle of nodes waiting to receive a message from each other, but no node sends a message because they are all waiting.
3. **Migration Deadlock:** Occurs when processes try to relocate or migrate execution to other nodes, but destination nodes are locked waiting for the migrating node's local files.

**c) There are a number of metrics that can be used to measure the performance of distributed systems. Explain any four (4) of these metrics. [8 Marks]**

1. **Response Time (Latency):** The total time taken from when a client submits a request to when they receive the first byte of the response.
2. **Throughput:** The total number of successful tasks, operations, or requests the system processes within a specific time unit (e.g., Transactions Per Second).
3. **Bandwidth Consumption:** The amount of network capacity consumed by internal node-to-node messaging overhead.
4. **Availability Uptime:** The ratio of time the system remains functional and accessible relative to its scheduled running time.

### QUESTION 3

**a) Scalability implies the ability to expand a distributed system in various dimensions. Discuss the different techniques that can be used to achieve this. [6 Marks]**
*(Note: This overlaps heavily with Question 1b. You can confidently reuse these standard architectural points)*:

1. **Horizontal Scaling (Scaling Out):** Adding more cheap, commodity machines to the cluster rather than buying one massive expensive mainframe.
2. **Data Partitioning (Sharding):** Breaking a massive central database down horizontally so different servers own separate subsets of the data rows.
3. **Load Balancing:** Deploying reverse proxies to automatically route incoming traffic evenly across a farm of identical backend workers.

**b) With the aid of a diagram, differentiate between two phase commit (2PC) and three phase commit (3PC) protocols. [4 Marks]**

* **Difference:** 2PC is a blocking protocol; if the coordinator crashes during the vote phase, participant nodes are left frozen in limbo. 3PC fixes this by adding an extra non-blocking intermediate step ("Pre-Commit") and uses timeouts so nodes can make a final decision even if the leader goes missing.

```
[2PC Flow]
Coordinator:   Can-Commit?  ---->  [Commit / Abort]
Participants:  Yes / No     <----  [Acknowledge]

[3PC Flow]
Coordinator:   Can-Commit?  ---->  Pre-Commit?  ---->  Do-Commit!
Participants:  Yes / No     <----  Ready        <----  Done

```

**c) Discuss the challenges of maintaining consistency in a distributed system. [4 Marks]**

1. **Network Partitions:** If a network break splits the system in half, it is impossible for nodes on both sides to sync up immediately, forcing a choice between accuracy or availability (CAP Theorem).
2. **Propagation Latency:** It takes physical time for an update on Server A to travel across the globe to Server B, creating a window where users might read stale, old data.

**d) Using examples explain the three (3) types of problems considered when implementing a distributed system fault tolerance strategy. [6 Marks]**

1. **Crash-Stop Failures:** A server completely halts and stops executing permanently (e.g., A power supply blows up on Node X).
2. **Omission Failures:** A node fails to perform an action, like failing to send or receive messages (e.g., A network router drops data packets due to buffer overflows).
3. **Byzantine (Arbitrary) Failures:** A node continues running but outputs corrupt, incorrect, or maliciously falsified data (e.g., A hacked server sending wrong balance figures to clients).

**c) Define Clock Synchronization and explain any clock synchronization algorithm of your choice. (6 Marks)**

* **Definition:** The mechanism used to coordinate and align the independent internal physical clocks of scattered nodes so they agree on a common system time.
* **Christian's Algorithm (Example):** A simple time-server mechanism. A client node sends a request message to a central Time Server machine. The server replies back with its accurate current time ($T_{server}$). The client calculates the round-trip network delay ($RTT$), halves it, and sets its local clock to:

$$\text{Time} = T_{server} + \frac{RTT}{2}$$

**a) A distributed transaction is supposed to satisfy the so called ACID properties. Explain what the ACID properties are. [4 Marks]**

### QUESTION 2

**a) Explain any three (3) types of security concerns that an ICT manager in large distributed organizations may deal with in the course of duty. [6 Marks]**

1. **Eavesdropping (Sniffing):** Attackers reading sensitive corporate data payloads moving across standard internet channels.
2. **Denial of Service (DoS):** Malicious flooding attacks designed to drown distributed servers in bogus traffic, locking out real employees.
3. **Identity Spoofing:** Rogue nodes or attackers faking credentials to gain entry into the internal network core.

**b) Describe the Optimistic Concurrency controls mechanisms used in Distributed Systems. [6 Marks]**
Optimistic Concurrency Control works on the assumption that data conflicts are rare.

1. **Read Phase:** Transactions read data and perform edits inside a private, isolated local workspace without locking the main database rows.
2. **Validation Phase:** Before saving, the system checks if other transactions changed that data while the transaction was working.
3. **Write Phase:** If no conflict is found, edits are permanently saved. If someone else changed the data, the transaction is discarded and tries again.

### QUESTION 3

**a) Discuss the importance of Distributed Data Architecture with reference to Distributed Systems. [8 Marks]**
Distributed Data Architecture establishes how data is divided and managed across servers. It is vital because:

1. It eliminates central database traffic jams.
2. It improves localized speed by storing regional user data on geographically closer servers.
3. It increases system survivability, ensuring that if Datacenter X floods, Datacenter Y still holds data copies to keep operations active.

**b) List and explain any four ideal features of a Distributed File System (DFS). [4 Marks]**

1. **High Availability:** Files are duplicated so they stay readable through disk or machine crashes.
2. **Location Transparency:** Users open files by name without needing to know which physical cabinet or state the hard drive sits in.
3. **Scalability:** The storage pool grows easily by plugging new hard drives into network racks without breaking the directory framework.
4. **File Locking:** Safe concurrency guards that prevent two users from overwriting the exact same file line simultaneously.

**c) Describe the Wait-die Vs Wound-wait in deadlocks in the context of Distributed Systems. [4 Marks]**
Both are timestamp-based methods used to prevent deadlocks by looking at transaction ages (Older transactions have lower timestamps):

* **Wait-Die Scheme (Non-preemptive):** If an old transaction wants a resource held by a young transaction, the old one is allowed to **wait**. If a young transaction wants a resource held by an old transaction, the young one **dies** (rolls back).
* **Wound-Wait Scheme (Preemptive):** If an old transaction wants a resource held by a young transaction, the old one **wounds** (forces rollback) the young one to claim the resource. If a young transaction wants a resource held by an old transaction, the young one is allowed to **wait**.

**d) Discuss the Scalability challenges in Distributed Systems. [4 M Marks]**

1. **Network Overhead Bottlenecks:** Adding thousands of nodes causes massive internal broadcast/sync traffic chatter, which can overwhelm network switches.
2. **Consistency Maintenance Tradeoffs:** As data spreads further out, keeping every node perfectly matched up in real-time slows down system writes drastically (CAP Theorem constraint).




Here is the straight-to-the-point, exam-focused breakdown of these core Distributed Systems concepts.

---

## 1. Foundation & Operating Systems

* **Distributed System:** A collection of independent computers that communicate over a network and appear to the end-user as a single, unified coherent system.
* **Network Operating System (NOS):** An OS where each computer runs its own local operating system. Users are fully aware of the distinct machines and must explicitly log in or copy files across remote nodes (e.g., standard Linux/Windows server setups).
* **Distributed Operating System (DOS):** A highly integrated OS that automatically manages resources across all connected machines. It completely hides individual hardware boundaries, acting as a single global OS for the entire cluster.

---

## 2. Core Architectural Principles

* **Transparency:** The ability of a system to hide its internal distribution and implementation complexities from users and applications (e.g., hiding where a file is physically stored).
* **Scalability:** The capacity of a system to handle growing amounts of work, or its potential to be enlarged to accommodate that growth (e.g., adding more nodes to handle more traffic).
* **Replication:** The process of copying and maintaining identical data or software components across multiple separate physical nodes to improve availability and read performance.
* **Data Management:** The structured practices used to store, partition, sync, and access datasets across scattered physical locations while keeping data accurate and retrievable.

---

## 3. Concurrency, Consistency & Control

* **Concurrency Control:** Mechanisms (like locking or timestamps) used to coordinate simultaneous operations on shared data, ensuring that parallel tasks do not corrupt the data.
* **Consistency:** A system rule ensuring that all distributed copies of a dataset match up and reflect the same updates at any given time, preventing users from reading stale or conflicting data.
* **Synchronization:** The coordination of processes or clocks across different nodes to agree on the correct order of events or absolute system time.
* **Deadlock:** A freeze state where a circular chain of two or more processes are permanently blocked because each is waiting for a resource held by the next one.

---

## 4. Dependability & Failure Concepts

* **Dependability:** The umbrella measure of a system's overall reliability, safety, security, and maintainability over time.
* **Availability:** The specific percentage of time that a system remains operational and accessible to process user requests (e.g., "99.9% uptime").
* **Reliability:** The probability that a system will continuously run and perform its functions without experiencing an absolute breakdown over a specified time interval.

### The Failure Chain

* **Fault:** A physical or logical defect, bug, or flaw within a component (e.g., a broken wire or a programming bug).
* **Error:** The manifestation of a fault. It is an incorrect internal system state caused directly by the fault (e.g., a corrupted bit in memory).
* **Failure:** The ultimate external event where the system completely stops delivering its expected service to the user because of an unhandled error.

---

## 5. Resiliency & Recovery

* **Fault Tolerance:** The design capability of a system to continue executing its tasks correctly even in the presence of hardware or software faults.
* **Redundancy:** The practice of building extra, identical components (hardware, software, or data copies) into the system to take over immediately if the primary components fail.
* **Data Recovery:** The techniques used to restore a system to a valid, clean, and consistent state after a crash occurs. This includes **Backward Recovery** (rolling back to a saved checkpoint) or **Forward Recovery** (skipping ahead to a new valid state).

---

## 6. Infrastructure & Storage

* **Security:** The protocols, encryption methods, and access controls used to protect distributed resources from unauthorized access, tampering, or malicious denial-of-service attacks.
* **Virtualization:** The software technology that abstracts physical hardware resources to create isolated, simulated Virtual Machines (VMs) or containers, allowing multiple workloads to share the same physical server.
* **File Systems (Distributed File System / DFS):** A file management system that allows client applications to access, modify, and store files physically located on remote servers exactly as if they were local files.
